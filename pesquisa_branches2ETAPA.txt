Q1: O branch, é basicamente uma copia do seu projeto que não afeta em nada, uma copia que serve como teste isolada, no próprio nome "branch" que traduz para "galho" imagina uma arvore com o tronco principal e um galho se movendo para esquerda porem o galho também é um tronco, o tronco original continua ali e não é afetado e pode continuar crescendo independemente, agora o galho é uma copia perfeita do tronco e pode ser alterada o quanto quiser que não afeta o tronco original.
Q2: O objetivo principal é prevenir erros e sempre ter uma versão que você possa voltar caso algo for errado, se tem um teste que quer fazer no projeto principal mas tem medo de dar errado, o branch serve para trabalhar sem medo.
Q3: A)Há um bug extremo em um programa que esta rodando no computador de clientes, mas modificar e tentar consertar o bug no código base não seria seguro, então você faz um branch idêntico e testa e se você achar a solução você substitui o código com o branch considering que é uma copia perfeita. B)Você pensa em uma maneira mais eficiente de rodar uma aplicação doque ela já e para não colocar em risco o resto do projeto você cria um branch onde você testa essa nova maneira. C)Você tem um bug no seu código porem você não sabe onde ele esta localizado, para testar você cria um branch e você vai testando e deletando linhas de código onde você acha que o bug pode estar.
